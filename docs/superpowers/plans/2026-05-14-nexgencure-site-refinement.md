# NexGenCure Site Refinement Plan

> 실행 기준: 이 계획은 넥스젠큐어 홈페이지를 일반적인 회사 브랜딩 사이트처럼 보이도록 다듬기 위한 작업 목록이다. 제품은 쇼핑몰처럼 판매하는 방식이 아니라, 회사의 현재 제품 라인업과 기술 기반을 소개하는 방향으로 정리한다.

## 목표

- 랜딩페이지 느낌보다 각 페이지가 독립적으로 완성된 회사 홈페이지처럼 보이게 한다.
- "개발품", "샘플", "나디온에서 선별"처럼 외부 출처나 임시 제작물로 보이는 표현을 줄인다.
- 넥스젠큐어가 뷰티·헬스케어 기업이며, 헬스케어 디바이스와 건강식품 라인업을 소개한다는 메시지를 명확히 한다.
- Company, Technology, Products, Materials, Contact 페이지가 각각 역할을 갖도록 내용과 구조를 보강한다.
- 모바일에서도 제목, 문단, 버튼이 잘리지 않도록 시각 QA를 진행한다.

## 작업 범위

- `index.html`: 메인 슬라이드 문구와 빠른 메뉴, 제품 소개 문구 정리
- `company.html`: 인사말, 사업영역, 핵심역량, 운영 기준 섹션 보강
- `technology.html`: 제조 기반, 연구개발, 제품화 프로세스 흐름 보강
- `products.html`: "개발 상품"이 아닌 "주요 제품 라인업" 관점으로 재정리
- `certification.html`: 실제 인증 자료가 없는 상태에 맞게 `Materials` 페이지로 전환
- `contact.html`: 상담 범위와 접수 후 흐름 보강
- `assets/css/styles.css`: 기업형 섹션, 정보 카드, 프로세스 리스트, 모바일 줄바꿈 보강
- `tests/site-content.test.mjs`: 회사 홈페이지 톤과 금지 표현 검증
- `scripts/verify-site.mjs`: 정적 검증 항목 업데이트

## 구현 단계

### 1. 회사 홈페이지 톤으로 문구 재정리

- 메인 카피를 "제품 개발품 소개"가 아니라 "뷰티·헬스케어 기업의 브랜드 신뢰" 중심으로 바꾼다.
- 제품 페이지의 제목과 본문을 `주요 제품 라인업` 기준으로 정리한다.
- 공개 HTML에서 아래 표현이 남지 않도록 검증한다.
  - 현재 개발 상품
  - 개발품입니다
  - 나디온
  - Nadyon
  - Certification 메뉴명

### 2. Company 페이지 보강

- 첫 화면은 회사 이미지와 신뢰감 있는 소개 문구로 유지한다.
- 하단으로 밀어낸 회사 기본 정보와 별개로, 본문에는 아래 섹션을 추가한다.
  - 사업영역
  - 핵심역량
  - 제조 기반
  - 제품 라인업
  - 연구개발
  - 운영 기준

### 3. Technology 페이지 보강

- 단순한 기술 키워드 나열이 아니라 제품화 흐름을 보여준다.
- `제품화 프로세스` 섹션을 추가하고 다음 흐름을 포함한다.
  - 기획
  - 제조
  - 연구개발
  - 자료화

### 4. Products 페이지 보강

- BS407과 MYONGMUNHWAN/명문염을 현재 소개 가능한 제품 라인업처럼 정리한다.
- 판매 버튼이나 장바구니 흐름은 넣지 않는다.
- 각 제품 카드에는 B2B 검토 관점의 포인트를 추가한다.
  - 브랜드 라인업 소개
  - B2B 제휴 검토용 정보 구성
  - 제품 이미지와 핵심 용도 중심 정리

### 5. Materials 페이지로 전환

- 실제 인증 자료가 없는 상태에서 `Certification`이라는 메뉴는 기대와 맞지 않으므로 `Materials`로 변경한다.
- 사업자 정보, 제품 자료, 기술·연구 자료를 검토 자료로 안내한다.

### 6. Contact 페이지 보강

- 문의 폼만 있는 화면이 아니라 회사 상담 페이지처럼 보이도록 한다.
- `상담 범위`를 추가한다.
  - 제품 라인업 상담
  - 기업 제휴 및 유통 검토
  - 회사소개서·제품자료 요청
- 접수 후 확인 가능한 자료와 상담 방향을 정리해 회신한다는 안내를 넣는다.

### 7. 검증

- `node --test tests/*.test.mjs`
- `node scripts/verify-site.mjs`
- 크롬 렌더링 캡처로 데스크톱과 모바일 화면을 확인한다.
- 모바일에서는 제목과 설명문이 오른쪽으로 잘리지 않는지 확인한다.

## 완료 기준

- 자동 테스트가 모두 통과한다.
- 금지 표현 검색에서 결과가 없어야 한다.
- 모바일 첫 화면에서 텍스트가 잘리지 않아야 한다.
- 전체 페이지가 회사 브랜딩 홈페이지처럼 읽혀야 한다.
